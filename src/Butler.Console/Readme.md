## asset

### Add

添加基金持仓，参数：fundcode 基金代码 cost 平均成本 share 份额

同一只基金可以添加多次，程序会自动计算出新的平均成本

当 share 为负值时，程序将处理为卖出基金持仓

有的平台计算的是摊薄成本，在盈利的情况下对持仓进行部分赎回，会导致摊薄成本变低、收益率上涨，因此建议使用平均成本数据，如果没有平均成本数据，可以通过交易记录计算出入金金额，平均成本=入金金额/持仓份额

### Remove

删除基金持仓，参数：fundcode

### Analyse

根据数据库中的基金持仓，进行数据分析：
- 分析股票、债券、现金占比
- 分析基金持仓收益

### Clear

清空基金持仓

## strategy

### GetAlphaBasedOnIndustry

获取基金剔除行业收益后的超额收益，参数：fundcode

在 asset.Analyse 中，对每个基金持仓的分析默认带有剔除行业收益后的能力分析

### GetFundTimingAbility

获取基金择时能力，参数：fundcode

执行前请先运行一次 data.Init

首次计算基金择时能力需要一至两分钟的时间，因此择时能力没有包含在 asset.Analyse 的默认分析中

计算过程中会使用到中证全指(000985.CSI)、中信金融(CI005917)、中信周期(CI005918)、中信消费(CI005919)、中信成长(CI005920)，但目前并未找到可以直接获取数据的数据源，因此需要用户自己从其他途径获取数据，目前程序仅支持 Choice 导出的数据，可参考 data 目录下的文件

## data

### Init

从 data 目录下的文件加载数据到 sqlite 中，例如 strategy.GetFundTimingAbility 就需要依赖 data 目录下的数据，因此需要先执行此命令预加载数据

## market

### Analyse

分析当前市场的情况，目前只能分析中信金融(CI005917)、中信周期(CI005918)、中信消费(CI005919)、中信成长(CI005920)的拐点

## ability

### AnalyseFund

分析基金能力，包含剔除行业收益后的能力分析以及择时能力的分析，参数：fundcode
